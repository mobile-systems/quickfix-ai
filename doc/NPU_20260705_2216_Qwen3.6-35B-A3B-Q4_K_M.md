## 📦 Вариант 1: Intel oneAPI (SYCL) — стабильный путь для Arc + NPU
Официальный Intel-бэкенд `llama.cpp` использует SYCL/DNNL, который автоматически распределяет нагрузку между Intel Arc GPU и NPU (AI Boost).

### 1. Установка зависимостей (Ubuntu 26.04)
```bash
# Intel oneAPI Base Toolkit (включает SYCL, DNNL, GPU/NPU драйверы)
wget -O- https://apt.repos.intel.com/intel-gpg-keys/GPG-Key-9AA14AE8 | sudo gpg --dearmor -o /usr/share/keyrings/intel-oneapi.gpg
echo "deb [signed-by=/usr/share/keyrings/intel-oneapi.gpg] https://apt.repos.intel.com/oneapi all main" | sudo tee /etc/apt/sources.list.d/oneAPI.list
sudo apt update
sudo apt install -y intel-oneapi-basekit intel-oneapi-compiler-dpcpp-cpp

# Ядро 6.8+ (в Ubuntu 26.04 уже есть)
uname -r  # должно быть ≥ 6.8

# Верификация SYCL-устройств
source /opt/intel/oneapi/setvars.sh
sycl-ls
# Ожидаемый вывод: gpu:0 Intel(R) Arc(TM) GPU, accelerator:0 Intel(R) NPU
```

### 2. Сборка `llama.cpp`
```bash
git clone --depth 1 https://github.com/ggerganov/llama.cpp.git
cd llama.cpp
mkdir build && cd build

# oneAPI SYCL бэкенд (включает GPU + NPU)
cmake .. \
  -DCMAKE_BUILD_TYPE=Release \
  -DGGML_SYCL=ON \
  -DGGML_METAL=OFF \
  -DGGML_CUDA=OFF \
  -DGGML_HIP=OFF \
  -DOPENBLAS=OFF \
  -DCMAKE_C_COMPILER=icx \
  -DCMAKE_CXX_COMPILER=icpx

make -j$(nproc)
```

### 3. Запуск и принудительное использование NPU
```bash
source /opt/intel/oneapi/setvars.sh

# Принудительно выбираем NPU-устройство
export SYCL_DEVICE_FILTER=accelerator
export ONEAPI_DEVICE_SELECTOR=sycl_level_zero

# Запуск инференса (ngl=100 = полная offload на Intel AI hardware)
./bin/llama-cli -m model.gguf \
  -ngl 100 \
  -t 16 \
  --ctx-size 4096 \
  --batch-size 512 \
  -p "System: You are a helpful trader assistant.\nUser: Explain latency arbitrage in HFT\nAssistant:"
```


### 2. Сборка `llama.cpp` с OpenVINO
```bash
cd llama.cpp/build
cmake .. \
  -DCMAKE_BUILD_TYPE=Release \
  -DGGML_OPENVINO=ON \
  -DGGML_METAL=OFF \
  -DGGML_CUDA=OFF \
  -DOPENBLAS=OFF

make -j$(nproc)
```

### 3. Запуск с NPU
```bash
export OPENVINO_NPU=1
export OPENVINO_ENABLE_NPU_FORCE=1

./bin/llama-cli -m model.gguf \
  -ngl 100 \
  -t 16 \
  --ctx-size 4096 \
  --batch-size 512 \
  -p "Prompt here..."
```

## 🛠 Тонкая настройка под Meteor Lake (Core Ultra 7 265T)
| Параметр | Значение | Обоснование |
|----------|----------|-------------|
| `-t` | `8..12` | NPU не выигрывает от >12 потоков, лучше оставить под OS/IO |
| `--ctx-size` | `4096` | NPU memory bandwidth ~100 GB/s, 8192+ даёт деградацию |
| `--batch-size` | `512` | Оптимум для SYCL/OpenVINO на AI Boost |
| `--numa` | `OFF` | `llama.cpp` уже делает NUMA-aware аллокацию |
| `--mlock` | `ON` | Фиксация модели в RAM, исключает page faults |
| `--numa` | `ON` | Включает NUMA-афинность (требует `numactl`) |

## 📌 Production-рекомендации
1. **NPU vs GPU**: Intel Core Ultra 7 265T имеет **Intel Arc GPU** и **AI Boost NPU (48 TOPS)**. Для `llama.cpp` стабильнее работает **Arc GPU via SYCL**. NPU эффективен при quantized models (Q4_0, Q5_K_M) и малых батчах.
2. **Quantization**: Используйте `q4_k_m` или `q5_k_m`. Intel NPU лучше работает с фиксированной точностью, чем с fp16/bf16.
3. **Ограничения**: Прямой NPU-бэкенд в `llama.cpp` experimental. Intel официально рекомендует `oneAPI SYCL` или `OpenVINO`. Для strict latency критичных сценариев рассмотрите `HuggingFace Optimum Intel` или `Intel AI Boost SDK`.
4. **Обновления**: Intel регулярно обновляет `llama.cpp` интеграцию. Следите за [Intel AI Toolkit](https://www.intel.com/content/www/us/en/developer/tools/oneapi/ai-kit.html) и [llama.cpp Intel backends](https://github.com/ggerganov/llama.cpp/wiki/Intel-AI-Toolkit).
